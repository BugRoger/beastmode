# Manifest Lifecycle

## Context
The manifest-file-management design splits manifest logic into two modules and renames directories: `pipeline/` becomes `state/` (gitignored manifests), `state/` becomes `artifacts/` (committed skill outputs). The github-no-for-real-sync design adds a sync-on-save hook to store.save() with re-entrancy guard.

## Decision
CLI creates manifest at first phase dispatch (design) via store.create(slug) before dispatching — manifest exists throughout the entire skill session. Enriched from output.json (auto-generated by Stop hook from artifact frontmatter) at each checkpoint. CLI is the sole mutator via two modules: manifest-store.ts (get, list, save, create, validate — sole fs accessor, with sync-on-save hook that triggers syncGitHub inside save() when github.enabled is true) and manifest.ts (enrich, advancePhase, regressPhase, markFeature, cancel, deriveNextAction, checkBlocked, shouldAdvance — pure functions returning new manifests). store.save() uses a module-level re-entrancy guard (_syncing flag) to prevent mutation write-back from re-triggering sync — simple, synchronous, safe in single-threaded Bun. Sync is fire-and-forget with warn-and-continue error handling — sync failures never block the save itself. Manifest location: `.beastmode/state/YYYY-MM-DD-<slug>.manifest.json` (flat-file convention, local-only, gitignored). CLI rebuilds from worktree branch scanning on cold start.

## Rationale
Two-module split separates filesystem concerns from business logic, making state machine functions testable with plain objects and no mocks. store.create() before dispatch ensures the manifest exists for the entire session, so post-dispatch always has a manifest to enrich. Sync-on-save guarantees every state change reaches GitHub without relying on callers to remember to sync. Re-entrancy guard prevents infinite loops when mutation write-back triggers a follow-up save. Directory rename makes names match contents: `artifacts/` for committed skill outputs, `state/` for gitignored pipeline state.
