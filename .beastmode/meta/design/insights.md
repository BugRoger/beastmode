# Design Insights

Emerging process patterns from design phases. Key themes: research-informed design decisions outperform brainstorming, fractal consistency across domains beats special-casing, HITL gate design requires structural enforcement and single-mechanism decisions, instruction visibility on critical paths demands visible markdown over HTML comments and inline over imported, scope management benefits from explicit deferral and per-instance enumeration, L0 content should be minimal (persona + map), agent organization follows observable runtime behavior, and external docs drift from internal knowledge hierarchy.

Insights span competitive analysis approaches, cross-session state management, instruction visibility patterns, scope management principles, L0 content scoping, agent file organization conventions, external documentation drift, and miscellaneous design patterns covering syntax design, boundary knowledge, and user interaction with design proposals.
