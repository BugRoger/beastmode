## Problem Statement

When the design phase exits without producing a PRD — whether the user quits gracefully, the session crashes, or the user simply abandons the interview — the CLI leaves behind orphaned artifacts: a worktree directory, a git branch, a manifest file that incorrectly advances to `plan` phase, and potentially a GitHub issue. These ghost epics pollute `beastmode status`, confuse the watch loop (which tries to dispatch plan on a PRD-less epic), and require manual `beastmode cancel <hex>` cleanup where the user has to remember an ephemeral 6-character hex slug.

## Solution

Detect missing output.json after the design session ends and automatically clean up all artifacts before returning. Two defense layers: phase.ts gates on output presence and runs cleanup before calling post-dispatch, and post-dispatch.ts adds a secondary guard that skips machine advancement for design phases with no output. Cleanup deletes the manifest entirely (not "cancelled" state), removes the worktree and branch, and closes any GitHub issue that was created.

## User Stories

1. As a user, I want to quit a design session early without leaving orphaned worktrees and manifests behind
2. As a user, I want the CLI to automatically clean up when a design session crashes (non-zero exit)
3. As a user, I want `beastmode status` to show no trace of abandoned designs (no ghost epics in plan phase)
4. As a pipeline operator, I want the watch loop to never dispatch plan on an epic that has no PRD
5. As a developer, I want `manifest-store.delete()` to be idempotent so cleanup is safe to retry

## Implementation Decisions

- Detection signal: `loadWorktreePhaseOutput(cwd, "design", epicSlug)` returning `undefined` means no PRD was produced — this is the sole signal, no secondary artifact check
- Primary gate in `phase.ts`: after `runInteractive()`, check for output.json before calling `runPostDispatch()`. If missing (either exit status), run cleanup and return
- Secondary gate in `post-dispatch.ts`: if phase is design and output is missing, return early without sending `DESIGN_COMPLETED` to the state machine
- Cleanup sequence: remove worktree (force, with branch deletion) → delete manifest file → close GitHub issue (warn-and-continue)
- Manifest fate: `store.delete()` removes the file entirely — no "cancelled" state, zero trace
- New `store.delete(projectRoot, slug)` function in `manifest-store.ts`: resolves path via existing convention, calls `fs.unlinkSync()`, returns boolean, idempotent (returns false if not found, does not throw)
- Partial artifacts: worktree removal is comprehensive — deleting the worktree directory removes any partially-written design artifacts. Only the manifest (in project root's `.beastmode/state/`) needs explicit deletion
- GitHub cleanup: if `config.github.enabled` and a GitHub epic issue was created during manifest seeding, close it as `not_planned` using same pattern as cancel command
- Both exit cases covered: non-zero exit (crash/Ctrl+C) and zero exit without output (graceful quit) both trigger the same cleanup path
- Log message: `"Design abandoned — cleaning up worktree and manifest"` — existing run log entry (already captured before cleanup) provides audit trail

## Testing Decisions

- Unit test `store.delete()` for: successful deletion, missing file returns false, idempotent repeated calls
- Integration test the design cleanup path: mock `runInteractive()` to return both `success` and `error` exit statuses without producing output.json, verify worktree/manifest/branch are removed
- Verify post-dispatch secondary guard: call `runPostDispatch()` with design phase and no output, verify `DESIGN_COMPLETED` is never sent to the state machine
- Verify state machine is never advanced: after cleanup, confirm no manifest exists and no machine state persists
- Prior art: cancel command tests (if they exist) provide patterns for worktree removal and manifest update assertions

## Out of Scope

- Resumable design sessions (user explicitly said no — abandoned means abandoned)
- Stop hook failure detection (false positive risk accepted — broken stop hook is a separate bug)
- Structured audit logging for abandoned designs (log message + run log is sufficient)
- Automatic retry of the design session
- Cleanup of designs abandoned in previous sessions (orphan detection is a separate feature)

## Further Notes

- The cmux/watch-loop path never dispatches design (interactive-only), so the secondary guard in post-dispatch is primarily for the ReconcilingFactory edge case
- The existing cancel command remains unchanged — it handles user-initiated cancellation of any phase, while this cleanup is automatic and design-specific

## Deferred Ideas

- Orphan detection: scan for manifests in design phase with no recent activity and offer cleanup
- Design resume: allow `beastmode design <hex>` to re-enter a worktree with partial progress
